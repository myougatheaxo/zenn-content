## この本を作ったきっかけ

「Androidアプリを作りたいけど、Kotlinって難しそう...」

「Jetpack Composeとか、なんか聞いたことあるけど、どこから始めればいいかわからない...」

そういう声、よく聞きます。みょうがも最初はそう思ってました。Kotlinの文法書を開いたら`data class`とか`sealed class`とか`coroutine`とかが出てきて、「あ、これは無理だな」って一瞬思ったんです。

でも**AIツールが登場してから、話が変わりました。**

Claude CodeやCursorに「ハビットトラッカーアプリを作って」と指示したら、MVVMアーキテクチャに沿った完全なKotlinコードを生成してくれるんです。しかも、Jetpack Composeで書かれた現代的なUIつきで。

みょうがはこれで気づきました。「コードを書く能力」と「アプリを作る能力」は、もはや別物だと。

---

## この本で学べること

この本では、以下の流れを一冊で完走します：

1. **AIツールの選び方** — Claude Code・Cursor・GitHub Copilotを比較して、自分に合ったものを選ぶ
2. **最初のアプリを作る** — ハビットトラッカーを題材に、AIと協力して実際に動くアプリを完成させる
3. **生成されたコードを理解する** — MVVM・Room・Composeの仕組みをざっくり掴む（丸暗記不要）
4. **カスタマイズする** — 色・名前・機能を変えて「自分のアプリ」にする
5. **Google Playに出す** — 申請から審査通過までのチェックリスト
6. **収益化する** — テンプレート販売・フリーランス受注で稼ぐ方法

## 必要なもの

| 項目 | 内容 |
|------|------|
| PC | Windows / Mac / Linux、どれでも可 |
| Android Studio | 無料でインストールできます |
| AIツール | Claude.ai（無料プランでも可）、または後述のツール |
| Androidスマホ | 実機テスト用（なければエミュレータで代用可） |
| Google Playアカウント | 公開するなら$25の登録料が必要 |

Kotlin・Java・プログラミングの事前知識は**不要**です。AIが書いてくれます。あなたがやることは「何を作りたいか」を伝えることと、生成されたコードをコピー&ペーストすることです（少し）。

## この本の読み方

各章は独立しているので、必要な章だけ読むこともできます。ただし、初めてアプリを作る人は第1章から順番に読むことを強くおすすめします。

コードサンプルが出てきたときは、**コピーするより先に「何をしているコードか」を1行だけ理解しようとしてください**。完全に理解できなくていいです。「あ、ここでデータを保存しているんだな」くらいのレベルで十分です。AIに「このコードの〇〇の部分は何をしているの？」と聞けば、すぐ教えてくれます。

では、始めましょう。

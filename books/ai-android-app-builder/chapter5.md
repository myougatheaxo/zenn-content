---
title: "Google Play公開チェックリスト — 申請から審査通過まで"
---

アプリが完成したら、いよいよGoogle Playに出す番です。

「審査が怖い」「何を準備すればいいかわからない」という声をよく聞きます。実際には、準備さえ整えれば審査はそれほど難しくありません。みょうがが実際に経験した流れをチェックリスト形式でまとめました。

---

## 事前確認: 公開前チェックリスト

### アプリの品質チェック

- [ ] エミュレータで全機能が動作する（API 26, 28, 33, 34 で確認推奨）
- [ ] 実機（Androidスマホ）でも動作確認済み
- [ ] クラッシュしないことを確認（いくつかの操作パターンを試す）
- [ ] 縦横回転（Orientation Change）でクラッシュしない
- [ ] バックボタンを押しても正常に終了する
- [ ] テキスト入力欄で長い文字列を入力してもクラッシュしない

### リリースビルドの準備

デフォルトのデバッグビルドはGoogle Playに出せません。署名付きのリリースビルドが必要です。

**1. 署名用キーストアの作成**

Android Studioのメニュー：`Build` → `Generate Signed Bundle / APK`

- **Android App Bundle** を選択（APKよりおすすめ）
- `Create new...` をクリック
- 設定：

| 項目 | 内容 |
|------|------|
| Key store path | 安全な場所に保存（なくすと詰む） |
| Password | 強いパスワードを設定して**必ず記録** |
| Key alias | アプリ名など分かりやすいもの |
| Key password | 別のパスワードでも同じでも可 |
| Validity years | 25以上推奨 |
| First and Last Name | 本名でなくてもOK |

**超重要**: キーストアファイル（.jks）とパスワードを**絶対に失くさないこと**。なくすと同じパッケージ名でアップデートできなくなります。Googleドライブや別のPCにバックアップを取ってください。

**2. リリースビルドの生成**

`Build` → `Generate Signed Bundle / APK` → 既存のキーストアを選択 → `release` を選択 → `Create`

`app/release/` フォルダに `app-release.aab` が生成されます。

## Google Play Console のセットアップ

**1. デベロッパーアカウント登録**

https://play.google.com/console にアクセスして登録：

- Googleアカウントが必要
- 登録料: **$25（一度だけ）**
- クレジットカード決済
- 反映まで数時間〜1日かかることがある

**2. 新しいアプリを作成**

「アプリを作成」をクリック：

| 項目 | 内容 |
|------|------|
| アプリ名 | ストアに表示される名前（最大30文字） |
| デフォルト言語 | 日本語向けなら「日本語」 |
| アプリかゲームか | 「アプリ」を選択 |
| 無料か有料か | ほとんどの場合「無料」（後でアプリ内課金も可） |

## ストア掲載情報の準備

### 必須項目チェックリスト

- [ ] **アプリ名**: 最大30文字。検索されやすいキーワードを含める
- [ ] **簡単な説明**: 最大80文字。最初の2行でユーザーが判断する
- [ ] **詳しい説明**: 最大4000文字。機能説明＋使い方＋FAQを含めると良い
- [ ] **アプリカテゴリ**: 「健康とフィットネス」「仕事効率化」など
- [ ] **タグ**: 最大5つ（関連キーワード）
- [ ] **メールアドレス**: サポート連絡先（公開される）
- [ ] **プライバシーポリシーURL**: 必須（後述）

### 必要な画像

| 種類 | サイズ | 枚数 |
|------|--------|------|
| アイコン | 512×512px PNG | 1枚 |
| フィーチャーグラフィック | 1024×500px PNG/JPG | 1枚 |
| スクリーンショット（スマホ） | 最小320dp幅 | 最低2枚、最大8枚 |

**アイコンの作り方（AI活用）：**

```
以下のアプリのアイコンを作って（SVGまたはPNG 512x512）：
- アプリ名: 習慣トラッカー
- テーマカラー: グリーン (#2E7D32)
- シンボル: チェックマーク付きのカレンダー
- 背景: 丸形（Android 13+の適応型アイコン対応）
- シンプルで視認性が高いデザイン
```

画像生成AIに投げればそれっぽいアイコンが作れます。みょうがはCanvaでも作っています（無料プランで十分）。

**スクリーンショットの取り方：**

エミュレータを起動した状態で、Android Studioの右パネルにカメラアイコンがあります。それをクリックするとスクリーンショットが撮れます。

## プライバシーポリシーの準備

アプリを公開するには**プライバシーポリシー**が必要です。

外部サーバーにデータを送らず、端末内のみで動くアプリ（ハビットトラッカーはこれ）でも必要です。

**最も簡単な方法: ジェネレーターを使う**

https://www.privacypolicytemplate.net/ か https://www.termsfeed.com/ などのサービスで無料生成できます。

AIへの指示でも作れます：

```
以下のAndroidアプリのプライバシーポリシーを日本語で作って。

アプリ名: 習慣トラッカー
パッケージ名: com.yourname.habittracker
開発者: [あなたの名前またはペンネーム]
連絡先: your@email.com

収集するデータ: なし（全データはデバイス内のみに保存）
第三者への提供: なし
広告: なし
```

生成されたテキストをGoogleドキュメントかGitHub Gistに貼り付けて、URLを取得します。そのURLをGoogle Play Consoleのプライバシーポリシー欄に入力します。

---

## コンテンツのレーティング設定

アプリの対象年齢・コンテンツに関する質問に答えます：

- 暴力的なコンテンツ: **なし**
- 性的なコンテンツ: **なし**
- ギャンブル要素: **なし**

正直に答えれば問題ありません。ハビットトラッカーなら「全年齢対象」になります。

---

## リリースの作成とアップロード

1. Google Play Console → 「本番環境」→「新しいリリースを作成」
2. `.aab` ファイルをアップロード
3. リリースノートを入力（例: 「初回リリース」）
4. 「次へ」→ 「送信」

---

## 審査について

送信後、**Googleの審査**が入ります。

- 初回審査: 通常1〜7日（最近は2〜3日のことが多い）
- アップデート: 通常数時間〜1日

**審査で落ちるよくある理由：**

| 理由 | 対策 |
|------|------|
| プライバシーポリシーがない | URLを必ず設定する |
| アプリが起動しない | リリースビルドで動作確認してから提出 |
| アイコンが規約違反（他社ロゴ使用など） | 自作かAI生成のオリジナル画像を使う |
| 説明文が詐欺的 | 実際の機能を正直に書く |
| 権限の過剰要求 | 使わない権限はManifestから削除 |

**審査通過後：**

Google Playに「公開」ボタンが表示されます。クリックすると数時間後に世界中のAndroidユーザーが検索で見つけられるようになります。

---

## 公開後のASO（アプリストア最適化）

公開してからが本番です。検索で見つかるようにするために：

**タイトルにキーワードを入れる：**
```
悪い例: 「習慣アプリ」
良い例: 「習慣トラッカー - 毎日の目標管理」
```

**説明文の最初の3行が命：**

多くのユーザーは「もっと見る」をタップしません。最初の80文字（簡単な説明）と説明文の最初の3行が勝負です。

**レビューを集める：**

初期レビューがないアプリはインストール率が低い。身近な人に使ってもらい、率直なレビューを書いてもらうのが最初の一手です。

---

## まとめ

- キーストアの保管は最重要（なくすと詰む）
- プライバシーポリシーのURLは必須
- スクリーンショットとフィーチャーグラフィックは丁寧に作る
- 初回審査は最大1週間みておく
- 公開後のASO（タイトル・説明文・スクリーンショット）で検索流入が変わる

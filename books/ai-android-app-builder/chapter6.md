Google Playにアプリを出せたなら、次は「これをどうやってお金に変えるか」です。

「趣味で作ったアプリで稼げるの？」と思うかもしれません。結論から言うと、**個人アプリ単体の広告収入だけで生活するのは非現実的**です。ダウンロード数が数十万を超えないと意味のある収益になりません。

でも、**アプリを作れるスキル自体を売る**なら話は別です。

---

## 収益化の3パターン

### パターン1: テンプレート販売

あなたがAIで作ったアプリは、そのまま**テンプレートとして販売**できます。

「え、AIが作ったものを売っていいの？」

はい。Claude CodeもCursorも、生成したコードの著作権はユーザー（あなた）に帰属します。AIはツールであり、Photoshopで作った画像を売るのと同じです。

**販売先の候補：**

| プラットフォーム | 特徴 |
|----------------|------|
| Gumroad | デジタル商品特化。手数料10%。海外ユーザーも多い |
| note | 日本語ユーザー向け。有料記事としてチュートリアル付きで販売 |
| BOOTH | 日本のクリエイター向けマーケット |

**テンプレートの価格設定：**

みょうがの経験則：

- 単体アプリテンプレート: $9.99〜$29.99
- バンドル（5〜8個セット）: $49.99〜$79.99

バンドルの方が圧倒的に売れます。「8個で$79.99」は1個あたり$10未満。買う側にとってお得感があるからです。

**何を含めるか：**

- ソースコード一式（Android Studioでそのまま開ける状態）
- README（セットアップ手順、カスタマイズ方法）
- カラーテーマの変更方法の説明
- スクリーンショット（商品ページ用）

### パターン2: フリーランス受託

「Androidアプリをゼロから作れます」と言えるだけで、クラウドソーシングサイトでは仕事が取れます。

**受託で稼ぐ流れ：**

1. Google Playに自作アプリを2〜3個公開する（実績として）
2. クラウドワークス・ランサーズ・ココナラに登録する
3. ポートフォリオとしてGoogle Playのリンクを載せる
4. 「Kotlin + Jetpack Composeでシンプルなアプリを作れます」と出品

**価格帯の目安：**

| 規模 | 相場 |
|------|------|
| シンプルなツールアプリ（電卓、タイマー等） | ¥30,000〜¥80,000 |
| データ管理系（家計簿、在庫管理等） | ¥80,000〜¥200,000 |
| API連携あり（天気、翻訳等） | ¥150,000〜¥300,000 |

AIを使えば、以前は1ヶ月かかったものが数日で作れます。つまり、**時給が跳ね上がる**ということです。

### パターン3: アプリ内広告（非推奨）

正直に言います。個人アプリの広告収入は**ほとんど稼げません**。

| DAU（日次アクティブユーザー） | 月間広告収入の目安 |
|------|------|
| 100 | ¥300〜¥1,000 |
| 1,000 | ¥3,000〜¥10,000 |
| 10,000 | ¥30,000〜¥100,000 |

DAU 1,000を達成するアプリはGoogle Playの上位5%に入ります。これは並大抵の努力では達成できません。

さらに、広告を入れると：
- ユーザー体験が悪くなる
- レビュー評価が下がる
- 権限が増える（INTERNET、AD_ID等）

みょうがのおすすめは、**広告なし・トラッキングなしを売りにして、テンプレート販売で稼ぐ**こと。「広告がないアプリ」は2026年の今、それだけで差別化になります。

## 実際の収益例

みょうがが8種類のテンプレートをバンドルで販売した場合のシミュレーション：

| 項目 | 数値 |
|------|------|
| バンドル価格 | $79.99 |
| Gumroad手数料 | 10% |
| 手取り/件 | $72 |
| 月間5件売れたら | $360/月 |
| 月間10件売れたら | $720/月 |

月5件は、技術記事やSNSからの導線がしっかりしていれば十分到達可能な数字です。

## 導線の作り方

テンプレートを作って置いておくだけでは売れません。**技術記事が最強の営業ツール**です。

**効果的な導線パターン：**

```
技術記事（Zenn/Qiita/Dev.to）
  → 「このアプリのテンプレートはこちら」
    → Gumroad商品ページ
      → 購入
```

記事のテーマ例：

- 「AIでAndroidアプリを47秒で作った話」
- 「無料アプリに年間5万円払ってた話」
- 「Claude Code vs Cursor vs Copilot 比較」
- 「コーディング経験ゼロで11分でアプリを動かす方法」

どの記事も最後に「テンプレートを公開しています」とCTAを置きます。読者が記事に価値を感じたら、テンプレートも買ってもらえる確率が上がります。

## 注意点

### 著作権とライセンス

- AIが生成したコードの著作権はあなたに帰属します
- ただし、テンプレートとして販売する場合は**ライセンスを明記**してください
- おすすめ: 個人利用・商用利用OK、再販売禁止
- Google Playの規約に違反するアプリは売らない

### 確定申告

年間20万円以上の利益（副業の場合）が出たら確定申告が必要です。Gumroadの売上は「雑所得」か「事業所得」として申告します。

### 詐欺にならない売り方

「プログラミング不要！AIが全部やってくれます！」— これは嘘ではないですが、誤解を招く表現です。

正確に言えば「コーディングの大部分をAIが担い、人間はAndroid Studioのセットアップと設定値の書き換えだけで済む」です。正直に伝えましょう。

## まとめ

- 個人アプリの広告収入だけでは稼げない
- **テンプレート販売**が最も再現性が高い
- フリーランス受託はポートフォリオがあれば受注可能
- 技術記事がテンプレート販売の最強の導線
- AIが作ったコードは販売OK（著作権はユーザーに帰属）

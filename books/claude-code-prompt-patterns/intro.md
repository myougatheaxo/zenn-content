やっほー、みょうがだよ。ウーパールーパーのVTuberで、普段はClaude Codeをガシガシ使って開発してるんだよね。

今日はぼくが実際の開発で使い倒してきた**プロンプトの「型」** を15パターンにまとめたよ。

Claude Codeって、ただ「これ作って」って言うだけでもそこそこ動いてくれるんだけど、**プロンプトの書き方を変えるだけで、出力の精度と速度が全然違う**んだよね。

このBookでは、以下の4カテゴリに分けて紹介するよ：

| カテゴリ | パターン数 | 内容 |
|---------|-----------|------|
| 基本パターン | 3つ | 日常的に使う基礎の型 |
| 効率化パターン | 4つ | 作業時間を半分にするテクニック |
| 品質管理パターン | 4つ | バグを減らす・品質を上げる |
| 高度なパターン | 4つ | マルチファイル操作やGit連携 |

各パターンには「いつ使う？」「プロンプト例（コピペ可）」「ポイント解説」「実行結果の例」が付いてるよ。

共通するのは「**Claude Codeに推測させない**」こと。ゴール、制約、手順を明確にするだけで、出力の品質は劇的に上がるんだよね。

さっそく第1章から始めよう。

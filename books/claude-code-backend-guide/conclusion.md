## Claude Codeと人間の役割分担

Claude Codeを使う上で大切なのは「何をAIに任せて、何を人間が判断するか」の線引きだ。

**AIに任せる（高速化できる）**:
- ボイラープレートコードの生成
- テストコードの作成
- ドキュメントの自動生成
- コードレビュー（一次チェック）
- パターンの実装（よく知られた設計パターン）

**人間が判断すべき**:
- ビジネスロジックの正確性
- セキュリティ要件の決定
- アーキテクチャの選択
- トレードオフの評価
- 本番環境への適用可否

## 明日から使えるプロンプトテンプレート

### 新機能実装時
```
「[機能名]を実装して。
技術スタック: [Node.js/TypeScript/Prisma/Redis]
要件:
- [要件1]
- [要件2]
セキュリティ: [特記事項]
テスト: 実装と同時にVitestでユニットテストも作成すること」
```

### レビュー依頼時
```
「[ファイル名/範囲]をレビューして。
重点的に見てほしい点:
- パフォーマンス（N+1クエリ、不要なDB呼び出し）
- セキュリティ（OWASP Top 10）
- エラーハンドリングの漏れ」
```

### リファクタリング時
```
「[ファイル名]をリファクタリングしたい。
現在の問題: [問題の説明]
目標: [達成したい状態]
制約: 既存のテストが全て通ること、外部インターフェースは変えない」
```

## 継続的な改善のサイクル

1. **実装** → Claude Codeに下書きを作らせる
2. **レビュー** → 人間が判断、Claude Codeが一次チェック
3. **テスト** → Claude Codeがテストコードを生成
4. **デプロイ** → CIで自動テスト
5. **モニタリング** → 問題発見 → 1に戻る

このサイクルを高速化するのがClaude Codeの本質的な価値だ。

## はじめに

Claude Codeに「ハビットトラッカーを作って」と指示して47秒。アプリは動いた。次は**Google Playに出す**番です。

「審査が怖い」「何を準備すればいいかわからない」という声が多いので、実際にアプリを公開した経験をもとに全手順をまとめました。

---

## 必要なもの

| 項目 | 備考 |
|------|------|
| Android Studio | 無料。リリースビルドの生成に必要 |
| Google Playデベロッパーアカウント | **$25（一度だけ）** |
| 署名用キーストア | Android Studioで作成 |
| プライバシーポリシーURL | AIで生成可能 |
| アイコン（512x512px） | AIで生成可能 |
| スクリーンショット（最低2枚） | エミュレータで撮影 |

---

## Step 1: リリースビルドの作成

デバッグビルドはGoogle Playに出せません。署名付きの**AAB（Android App Bundle）**を作ります。

Android Studio → `Build` → `Generate Signed Bundle / APK` → **Android App Bundle**を選択

### キーストアの作成

初回は `Create new...` をクリック：

| 項目 | 設定 |
|------|------|
| Key store path | 安全な場所に保存 |
| Password | 強いパスワード |
| Key alias | アプリ名など |
| Validity years | **25年以上** |

:::message alert
**キーストアファイル（.jks）とパスワードは絶対になくさないこと。** なくすと同じパッケージ名でアップデートできなくなります。Googleドライブにバックアップを取ってください。
:::

`release`を選択 → `Create` → `app/release/app-release.aab` が生成されます。

---

## Step 2: Google Play Consoleのセットアップ

https://play.google.com/console にアクセスして登録。

**アプリの作成：**

| 項目 | 内容 |
|------|------|
| アプリ名 | 最大30文字。検索キーワードを含める |
| デフォルト言語 | 日本語 or English |
| アプリかゲームか | アプリ |
| 無料か有料か | ほとんどの場合「無料」 |

---

## Step 3: ストア掲載情報

### 必須チェックリスト

- [ ] **アプリ名**: 最大30文字（例: 「習慣トラッカー - 毎日の目標管理」）
- [ ] **簡単な説明**: 最大80文字。ここで離脱されるので命がけで書く
- [ ] **詳しい説明**: 最大4000文字。機能+使い方+FAQ
- [ ] **カテゴリ**: 「健康とフィットネス」「仕事効率化」等
- [ ] **メールアドレス**: サポート連絡先（公開される）

### 必要な画像

| 種類 | サイズ | 枚数 |
|------|--------|------|
| アイコン | 512×512px PNG | 1枚 |
| フィーチャーグラフィック | 1024×500px | 1枚 |
| スクリーンショット | 最小320dp幅 | 2〜8枚 |

**AIでアイコンを作る方法：**

画像生成AIに「チェックマーク付きカレンダーのアイコン、グリーン基調、512x512、丸形」と指示すれば作れます。Canva（無料プラン）でも十分。

**スクリーンショット**はAndroid Studioのエミュレータ右パネルのカメラアイコンで撮影。

---

## Step 4: プライバシーポリシー

端末内のみで動くアプリでも**必須**です。

AIへの指示で生成：

```
以下のAndroidアプリのプライバシーポリシーを作って。
- アプリ名: 習慣トラッカー
- 収集データ: なし（端末内のみ保存）
- 第三者提供: なし
- 広告: なし
```

生成テキストをGoogleドキュメントかGitHub Gistに貼り付け、URLを取得。

---

## Step 5: レーティング設定

暴力・性的・ギャンブル要素について質問に答えるだけ。ツールアプリなら全て「なし」→ **全年齢対象**になります。

---

## Step 6: アップロードと審査

1. 本番環境 → 新しいリリースを作成
2. `.aab` ファイルをアップロード
3. リリースノートを入力
4. 送信

**審査期間：**

| 種類 | 期間 |
|------|------|
| 初回審査 | 1〜7日（最近は2〜3日が多い） |
| アップデート | 数時間〜1日 |

---

## 審査落ちの理由TOP5と対策

| # | 理由 | 対策 |
|---|------|------|
| 1 | プライバシーポリシーがない | URLを必ず設定 |
| 2 | アプリが起動しない | リリースビルドで動作確認 |
| 3 | アイコンが規約違反 | 自作かAI生成のオリジナルを使う |
| 4 | 説明文が詐欺的 | 実際の機能を正直に書く |
| 5 | 権限の過剰要求 | 不要な権限をManifestから削除 |

:::message
AI生成アプリは不要な権限（INTERNET、AD_ID等）を含まないので、権限関連で審査落ちするリスクが低いです。
:::

---

## 公開後のASO（アプリストア最適化）

**タイトルにキーワードを入れる：**

```
✗ 「習慣アプリ」
○ 「習慣トラッカー - 毎日の目標管理」
```

**説明文の最初3行が命。** 多くのユーザーは「もっと見る」をタップしません。

**初期レビューを集める。** レビューゼロのアプリはインストール率が極端に低い。まず身近な人にお願いする。

---

## まとめ

- キーストアの保管が最重要
- プライバシーポリシーURLは必須
- 画像（アイコン、スクリーンショット）は丁寧に作る
- 初回審査は最大1週間
- AI生成アプリは権限が少ないため審査に有利

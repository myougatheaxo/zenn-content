## トークンコストが膨らむ原因

Claude Codeを使い始めると、コストが予想外に高くなることがある。主な原因:

1. **不必要に長いCLAUDE.md** → 毎回コンテキストに含まれる
2. **大きなファイルを丸読み** → 必要な部分だけ読めばいい
3. **Opusを何でも使う** → HaikuやSonnetで十分なタスクが多い
4. **繰り返し同じ調査をする** → キャッシュ・メモリを使う

---

## 削減テクニック1: CLAUDE.mdを最小化する

CLAUDE.mdはすべての会話コンテキストに含まれる。長ければ長いほどコスト増。

**Before** (300行のCLAUDE.md):
```markdown
## 環境情報
OS: Windows 11
Python: 3.13
Node.js: 20.x
...（詳細なセットアップ手順50行）
...（コーディング規約100行）
...（ツール使い方50行）
```

**After** (50行以内に凝縮):
```markdown
## コア規約（必須）
- 最小限の変更のみ
- テスト変更後は既存テスト実行
- 参照: `◆knowledge/patterns.md`

## 技術スタック
Python 3.13 / Node 20 / Windows 11
```

詳細は別ファイルに書いて「参照してください」と書くだけでいい。AIが必要な時だけ読みに行く。

---

## 削減テクニック2: モデルを使い分ける

| タスク | Opus | Sonnet | Haiku |
|--------|------|--------|-------|
| ファイル検索 | ❌高すぎ | △ | ✅ |
| 実装 | △ | ✅ | ❌遅い |
| 設計判断 | ✅ | △ | ❌ |
| grep/検索 | ❌ | ❌ | ✅ |

Haikuはトークン単価がOpusの1/15程度。単純な検索・一覧系はすべてHaikuに任せる。

---

## 削減テクニック3: /clearを活用する

長い会話になるとコンテキストが肥大化してコスト増。

```bash
# 作業が一段落したら
/clear

# 前の作業の状態を引き継ぐ場合はメモリに残しておく
```

サブエージェントは自動的に独立したコンテキストになるので、メインコンテキストを汚染しない。

---

## 削減テクニック4: Grep/Globを積極使用

```
❌ 「src/以下のすべてのファイルを読んで」
✅ 「Grepで `UserService` クラスを探して」
```

Readツールで大きなファイルを丸読みせず、必要な行番号にオフセットして読む。

```python
# ❌ ファイル全体読み込み (2000行)
# ✅ 特定行だけ読む
read(file_path="src/service.py", offset=150, limit=50)
```

---

## 削減テクニック5: キャッシュ前提のメモリ設計

調査結果をメモリファイルに保存 → 次回以降は読むだけ。

```markdown
# memory/api-patterns.md
## UserAPI エンドポイント一覧 (2026-03-11 調査済み)
- GET /users/:id → UserController.getById (src/controllers/user.ts:42)
- POST /users → UserController.create (src/controllers/user.ts:67)
```

同じ調査を2度やらない。これだけで繰り返し作業のコストがゼロになる。

## 実際の削減効果

| 施策 | コスト削減率 |
|------|------------|
| CLAUDE.md最小化 | 20-30% |
| Haiku活用 | 15-25% |
| /clear習慣化 | 10-15% |
| Grep/Glob優先 | 10-20% |
| メモリ活用 | 5-15% |
| **合計** | **60-70%** |

---

## まとめ

コスト削減は「読む量を減らす」「安いモデルを使う」「繰り返し作業をキャッシュする」の3原則。CLAUDE.mdを50行以内に保つだけで即効性がある。

## はじめに

「無料で使えるならそれでよくない？」

そう思ってた。2026年の最新データを見るまでは。

この記事では、無料アプリの「隠れたコスト」を数字で検証し、代替手段としてAI生成アプリを紹介します。

## 無料アプリの3大コスト

### 1. データ収集コスト

2026年の調査データ：

| 指標 | 数値 |
|------|------|
| トラッカー内蔵率 | 94% |
| 年間データ収集量 | 4.2GB/ユーザー |
| 共有先の第三者数 | 平均5.3社 |
| 不必要な権限要求率 | 45% |

収集される情報：
- デバイスID、広告ID
- 位置情報（バックグラウンドでも）
- 連絡先リスト
- ブラウジング履歴
- アプリ使用パターン（起床時間、通勤経路）

### 2. 時間コスト

| 指標 | 数値 |
|------|------|
| 1日あたりの広告表示 | 30回以上 |
| 広告による時間損失 | 22分/日 |
| 年間の時間損失 | 134時間 |

134時間 = **5.5日分**の時間を広告に費やしている。

### 3. 金銭コスト

「無料」なのにお金がかかる？

| 費目 | 年間コスト |
|------|----------|
| アプリ内サブスクリプション | $230/年（平均） |

## 自分で作ると何が変わるか

### Roomデータベースの例

AIが生成したKotlinコード：

```kotlin
@Entity(tableName = "habits")
data class Habit(
    @PrimaryKey(autoGenerate = true) val id: Int = 0,
    val name: String,
    val createdAt: Long = System.currentTimeMillis(),
    val streak: Int = 0
)
```

注目点：**INTERNET権限がない**。データは端末内に保存される。外部に送信されない。

### AndroidManifest.xmlの比較

```xml
<!-- 無料アプリ（典型） -->
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
<uses-permission android:name="android.permission.READ_CONTACTS"/>
<uses-permission android:name="android.permission.AD_ID"/>

<!-- 自作アプリ -->
<!-- 権限なし。ローカルで完結 -->
```

## 実際に自作する方法

必要なもの：
- PC（任意のOS）
- Android Studio（無料）
- Androidスマホ
- 11分の時間

手順：
1. テンプレートをダウンロード
2. `strings.xml`でアプリ名を変更
3. `Theme.kt`でカラーコードを変更
4. Android Studioでビルド
5. USBでスマホにインストール

コーディング経験は不要。設定値を書き換えるだけ。
